## Summary

Build the complete Fast Printing & Packaging platform across three connected surfaces:

1. **Backend API** (Node.js 20 + TypeScript + Express + PostgreSQL + Prisma)
   вҖ” Authentication, product catalogue, cart, orders, payment, quotes, contact, admin CRUD,
   and Cloudinary image management.

2. **Flutter Mobile App** (Android)
   вҖ” Customer-facing app: browse, wishlist, cart, order, payment proof upload, quotes, contact.

3. **React Admin Dashboard** (Web SPA)
   вҖ” Admin panel: order verification, content CRUD with image upload, quote/contact
   inbox, user management, dashboard metrics.

All three surfaces share a single backend via `/api/v1` REST endpoints. Auth is Google
OAuth вҶ’ backend-issued JWT. Images are uploaded via the backend to Cloudinary and served
to Flutter via CDN URLs. No Firebase. All infrastructure is free-tier.

## Constitution Re-Check (Post-Design)

- вң… **Art. I**: All free-tier. No paid services added.
- вң… **Art. II**: No Firebase. No `google-services.json`. Direct Google OAuth only.
- вң… **Art. III**: `google-auth-library` verifies idToken server-side. JWT 64-char secret. Payment numbers in `.env` only. JWT in `flutter_secure_storage`.
